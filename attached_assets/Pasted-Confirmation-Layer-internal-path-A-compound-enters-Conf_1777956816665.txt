Confirmation Layer — internal path
A compound enters Confirmation carrying an InVitroPackage from L1: hit identity, primary assay results, ML feature vectors, and L1's candidate ranking with confidence. The layer's job is to either kill it cheaply, send it to Animal with a typed evidence package, or route it to override. Seven phases.
1. Ingest and lock. Confirmation accepts the InVitroPackage, validates the schema, and freezes an evidence pointer. Every artifact Confirmation emits references this pointer for L1 provenance. If the package is missing required fields (no ML feature vectors, no primary assay results), Confirmation refuses entry — that's an L1 contract violation, not a Confirmation problem.
2. In-silico sub-track runs first. ADMET, PBPK, off-target, dose-response, and the §6.1.a CYP-Transporter / §6.1.b clearance ML predictions all run here. Output is a per-compound confidence vector across each prediction class plus a typed PreConfirmationDirective for the ex-vivo track: the prioritized assay list, expected response ranges, the falsifiers that would invalidate the in-silico predictions, and recommended dose ranges. The directive is the contract — ex-vivo cannot start without one.
3. In-silico gate (the budget gate). Compound passes if minimum confidence is met across critical prediction classes (configurable per program, defaults conservative). It fails if confidence is below threshold and no falsifier path is defined. It is overridable only if a human or policy explicitly accepts a named falsifier — and that override carries {actor, reason, falsifierAccepted, l1EvidenceCited} so Phase1Planner can read it later. Compounds failing this gate without override do not proceed to ex-vivo. This is what protects the wet-lab budget and is the single most important property of the whole layer.
4. Ex-vivo sub-track consumes the directive. Assay selection is directive-driven, not free-form. Hepatocyte clearance confirms §6.1.b. CYP inhibition panel confirms §6.1.a. 3D tissue toxicity confirms predicted safety profile. Permeability assays confirm absorption. Each assay emits a result tested against the directive's expected range and falsifier set. An ex-vivo run that doesn't reference a directive is rejected at write-time — same enforcement model as L1 schema validation.
5. Falsifier check. For each compound, did any directive-flagged falsifier trigger in ex-vivo? If yes, the in-silico prediction for that domain is falsified, not "low confidence" — they are different states with different downstream consequences. Falsified predictions are marked explicitly and do not silently average with ex-vivo into a softer joint number. This is the property that lets Phase1Planner make defensible IND-gate decisions later, including the MABEL bias case.
6. Joint output synthesis. For each domain that's a joint L1+L2 output (CYP-Transporter, clearance-fractions, off-target, dose-response): when in-silico and ex-vivo agree within tolerance the joint signal is emitted with high confidence; when they disagree ex-vivo wins, the in-silico prediction is marked falsified for that domain, and the joint output carries a confidenceCaveat; when ex-vivo is missing the corresponding assay the joint output is marked partial with an mlOnlyEvidence flag and no silent fallback to in-silico. These land at confirmation.jointOutputs.* with explicit mlEvidenceRef pointers up to L1 and kineticEvidenceRef (or equivalent) pointers down to the specific ex-vivo records that produced them.
7. Exit gate to L3 Animal Cohort. Compound passes if the in-silico gate passed or was overridden, the ex-vivo gate passed with no unhandled falsifiers, and all joint outputs are reconciled or explicitly marked partial. The ConfirmationPackage emitted to L3 carries per-track evidence blocks ({evidence, overrides, confidence, gateStatus, falsifiers, emittedAt, emitterEngine}), the joint output blocks, the directive itself (so animal study design can reference what was predicted vs. confirmed), and the full override audit trail.
Edge cases worth pre-deciding now, not later. When in-silico has no prediction because of a training gap, don't degrade to "low confidence" — emit a directiveGap flag, let ex-vivo run a broader assay panel, and mark the joint output mlGap. When an ex-vivo assay fails technically, apply a bounded retry policy and on exhaustion record assayFailureRecord with partial confirmation rather than blocking. When a compound is well-characterized enough that ex-vivo would be wasted (close prior art, known pharmacology), allow an explicit exVivoSkipped{reason, l1MlConfidenceThresholdMet, priorArtRef} override — Phase1Planner sees the skip and biases conservative on dose.
External face. Partners see one gate ("Confirmation passed for compound X with confidence Y") and one typed package handed to Animal Cohort. The sub-track structure is internal infrastructure — visible to teams who need to debug, override, or attribute confidence, transparent to BD.
The risk this design specifically defends against is the one you flagged — sub-tracks drifting back into one blob within a quarter. Three properties carry that load: the ex-vivo-cannot-start-without-directive enforcement, the falsified-vs-low-confidence distinction in the falsifier check, and the no-silent-fallback rule on joint outputs. If any of those gets relaxed in v2.0.x for "convenience," the design is gone and we're back to where we started.
