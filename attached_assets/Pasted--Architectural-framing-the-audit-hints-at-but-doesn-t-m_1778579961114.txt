
Architectural framing the audit hints at but doesn't make explicit
Promote anti-fabrication to a write-time invariant the way v2.0.0 Confirmation already does. The nine red flags are functionally invariant violations — Unknown Target Kd values, lung-risk noise, the EGFR similarDrugs fallback, fake off-target IC50s. The ML pipeline already carries the right envelope (source: "ml" | "fallback") and cascadeGate.ts has the concept (tier, note). Promote it to a type-level requirement everywhere: any numeric L1 output must carry { value, source, note } or the build refuses. That converts §9.8 from an aspiration ("surface gate notes on main panels") into something the type system enforces.
Unify the two Go/No-Go authorities. inVitroResults.overallRisk and simResults.goNogo can disagree on the same molecule and the audit notes this without resolving it. Pick the cascade gate as the sole gate; demote the other two to inputs. Today the user can be shown "GO" by one pipeline and "WATCH" by the other on the same screen.
Wire cascade-gate notes into HAIOps Gate Service. Per the locked-in contract decisions from May 6, the module owns falsifier definitions and HAIOps owns registry/dispatch/audit. The cascade-gate note strings ("Pharmacophore + jitter from hardcoded potency range") are de-facto falsifier definitions buried in one file — register them with HAIOps so they're auditable and dispatchable system-wide, not local lore.
Red flags missing from §7
The confidence score is calibration-free. Rendering "Confidence Score: 78%" implies probabilistic calibration that the weighted-sum formula doesn't have. Re-label as a composite heuristic score, or replace with a bootstrap-over-jitter estimate with Brier-score validation.
selectivity.totalTargets = 50 + jitter*50 + heavyAtoms renders as "tested against ~100 targets." That's a fabricated experimental claim, equivalent severity to the off-target IC50 issue. Remove it — you can't emit a "tested against N" count when N actual screens were run.
potencyClass ("High / Moderate / Low") is TA-conditional but unlabelled. Under TA = Default (which the α-tocopherol-fragment case will hit), the threshold convention is undefined and the label is meaningless. Always show the convention.
The hardcoded patchClampResult strings deserve a numbered slot in the red-flag list, not just a paragraph mention — same severity class as the others.
Cleanup order — reorder by demo embarrassment, not by LOC
§9 is engineer-friendly (smallest change first). Reorder by what would do the most damage if a regulator or partnered biotech ran α-tocopherol-fragment live: fake named off-target IC50s first ("vitamin E hits PIM1 at X nM"), then off-catalog binding Kd, then the EGFR similarDrugs fallback, then the fake totalTargets, then lung-organ noise. The misnamed qed/saScore and the dead genoRisk come later — they're labelling problems, less viscerally damaging in a live demo.
Section 0.5 (ML pipeline) — what to add
None of the six models declares an applicability domain. Without that, α-tocopherol-fragment gets numeric DILI / hERG / solubility predictions rendered on the same UI surface as in-distribution molecules. At minimum surface Tanimoto-to-training-nearest-neighbor for the Morgan-FP RFs, and ensemble disagreement / softmax entropy for ADMET-AI's Chemprop ensemble, as an in_distribution_band: high|medium|low flag visible next to each value.
Be loud about sample sizes. Binding-ML at ~125 pairs and Protein-ML at ~80 + ~80 are tiny by ML standards. The honesty contract should surface held-out test AUC / RMSE next to each model name, not just inference_time_ms. If you can't say it, that's already a result.
Specify the CardioTox ↔ ADMET-AI hERG consensus threshold. "Aligned / Divergent / Consensus" is mentioned without definition; pick an explicit rule (e.g., same qualitative class → Aligned; probabilities within ±0.15 → Consensus; opposite verdict → Divergent).
DILI-ML's honest scope is binary risk; the cascade gate uses it as an L1 hepatic override and as L3 Stage 2 input. Add a test that the gate never escalates heuristic severity or mechanism flags from a DILI-ML signal alone — only its probability. Per your memory, safety outputs are heuristic-only and require a reviewer gate before animal advancement; this is the same line.
Document what ADMET-AI's curated KEY_ENDPOINTS discards. If a researcher asks "what about LD50_Zhu?", you want a one-line answer rather than a code dive.
Machine-enforce the audit
Promote α-tocopherol-fragment from a manual cross-check into a fixture-based regression suite. A small panel of "should produce no binding / no off-targets / no analogues" molecules — chromanol head, a sugar, a fatty acid, a noble-gas-like solvent surrogate. For each, assert that binding, similarDrugs, offTargetHits, doseResponse.ic50_nM, and patchClampResult either return null or render their explicit fallback banner (snapshot test). Run on every PR. The audit stops being recurring lore and becomes a contract.
L2/L3 handoff typing
The audit is L1-only and says so. Before the L2 and L3 audits land, type the L1→L2 handoff so the L2 consumer can refuse heuristic-only inputs on safety-critical paths. Otherwise every L1 fabrication propagates into the Confirmation layer's animal-advancement decision and the L1 cascade gate's tier: heuristic tagging buys you nothing downstream.
Two cheap code wins worth front-loading
genoRisk should be wired to a real Ames-style SMARTS panel using the existing bioactivationEngine.ts pattern (aromatic amines, nitroso, epoxides, Michael acceptors of the alkylating-DNA flavor). Replacing dead code with a working structural screen is a stronger fix than deletion — and it's a few hours of work given the existing engine.
qed and saScore should be vendored from their published reference implementations (Bickerton 2012 for QED; Ertl-Schuffenhauer 2009 for SA score). Both are ~150 lines of mechanical port and remove two red flags at once while keeping the panel labels honest.
