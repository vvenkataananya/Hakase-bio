## Tier 1: PK-Sim/MoBi — The Open-Source Foundation

### What It Does
Whole-body PBPK modeling with **built-in animal physiology libraries** for mouse, rat, dog, monkey, and human. Explicitly designed for **cross-species extrapolation** using your In Vitro parameters .

### Automation Architecture

**R-Based Pipeline (Recommended)**
```r
# Core packages from Open Systems Pharmacology
library(ospsuite)        # Load/simulate PK-Sim/MoBi models
library(tlf)             # Standardized Tables/Listings/Figures
library(ospsuite.reportingengine)  # Automated reports
library(ospsuite.parameteridentification)  # Model fitting
```

**Key Automation Features** :
- **Command Line Interface (CLI)**: Batch process multiple PK-Sim projects without GUI
- **PKML Format**: XML-based model exchange between PK-Sim and MoBi
- **Qualification Framework**: Automated validation of simulation scenarios for regulatory confidence

**Batch Workflow Example**:
1. Export your In Vitro parameters (logP, solubility, permeability, Clint, fu)
2. Load species-specific template (rat.pkml, dog.pkml, monkey.pkml)
3. Set compound parameters via API
4. Run population simulation (10-100 virtual animals)
5. Extract PK parameters (AUC, Cmax, Tmax, CL, Vdss)
6. Generate qualification report

### Integration Points
| Your Data | PK-Sim Input | Animal Output |
|-----------|--------------|---------------|
| Caco-2 Permeability | Effective permeability (Peff) | Fraction absorbed (Fa%) |
| Hepatocyte Clint | Intrinsic clearance | Hepatic clearance, Bioavailability |
| Plasma Protein Binding | fu (unbound fraction) | Free drug concentration, CNS penetration |
| Solubility/pKa | Dissolution parameters | Absorption profile, Tmax |
| Blood-Brain Barrier (In Vitro) | Permeability-limited brain model | CNS exposure, behavioral target engagement |

## Tier 2: Simcyp Animal Simulators — Commercial Precision

### What It Does
**Simcyp Animal** offers PBPK/PD modeling for rats, dogs, monkeys, and mice with **virtual population variability** and advanced organ models (5-compartment brain, permeability-limited distribution) .

### Automation Architecture

**R Package Integration** :
```r
# Simcyp R package workflow
library(SimcypR)

# 1. Initialize connection to Simcyp engine
Initialise()

# 2. Load pre-built animal workspace (rat/dog/monkey)
SetWorkspace("rat_adme.wksz")

# 3. Modify parameters using Tag IDs (from GUI right-click)
SetParameterValue(tagID = "Compound.Lipophilicity", value = 2.4)
SetParameterValue(tagID = "Compound.Solubility", value = 0.05)

# 4. Run simulation
Simulate()

# 5. Extract results from .db file
GetCompoundResults_DB("simulation.db")
GetProfile_DB("simulation.db", "Plasma")
```

**SimRFlow: High-Throughput Automation** :
- **No compound limit** (vs. 553 in EPA httk)
- **Batch processing** via CSV input files
- **Species**: Currently human; rat/mouse/dog expansion in next version
- **Administration routes**: Oral, dermal, IV bolus
- **Population simulations**: Up to 1,000 virtual subjects

**Input CSV Format** (Minimal Requirements):
```csv
CompoundName,SMILES,InChIKey,Dose,AdministrationRoute,NumberOfIndividuals,SimulationDuration
YourMolecule,CC(C)Cc1ccc(C(C)C(=O)O)cc1,InChIKey=... ,25,Oral,10,24
```

### Advanced Features

## Tier 3: Validation Layer — ToxRefDB & ToxCast

### ToxRefDB (EPA)
**What it is**: Curated animal toxicity study results from EPA pesticide testing—chronic, subchronic, developmental, reproductive toxicity in rats, mice, rabbits, dogs .

**Access Method**: Through **EPA CompTox Dashboard** (web interface + API)
- Search by chemical structure, CAS, or name
- Retrieve NOAEL, LOAEL, critical effects, study types
- **Use case**: Validate your PBPK predictions against historical animal outcomes

**Integration Workflow**:
1. Run PK-Sim/Simcyp to predict tissue exposure (Cmax, AUC)
2. Query ToxRefDB for structurally similar compounds
3. Compare predicted exposure vs. historical toxic doses
4. Flag if predicted Cmax approaches historical LOAEL

### ToxCast/Tox21 (EPA/NIH)
**What it is**: High-throughput screening data across hundreds of *in vitro* assays designed to predict *in vivo* animal toxicity .

**Integration for Animal Prediction**:
- Feed your molecule's ToxCast assay profile into **IVIVE models**
- Predict probability of liver injury, cardiotoxicity, developmental effects
- Use as **early safety filter** before PBPK simulation

## Recommended Integration Pipeline

### Phase 1: Automated In Vitro → Animal PK (Week 1-2)

```python
# Pseudocode for integrated workflow
import ospsuite  # or SimcypR via rpy2

def in_vitro_to_animal_pk(compound_data, species="rat"):
    """
    compound_data: dict with solubility, permeability, 
                   metabolic_clint, protein_binding, logP, pKa
    species: "rat", "dog", "monkey", "mouse"
    """
    
    # 1. Load species template
    model = ospsuite.load(f"templates/{species}.pkml")
    
    # 2. Set compound parameters from In Vitro data
    model.set_parameter("Lipophilicity", compound_data['logP'])
    model.set_parameter("Solubility", compound_data['solubility'])
    model.set_parameter("SpecificIntestinalPermeability", compound_data['permeability'])
    model.set_parameter("IntrinsicClearance", compound_data['metabolic_clint'])
    model.set_parameter("FractionUnbound", 1 - compound_data['protein_binding'])
    
    # 3. Run virtual population (10-50 animals)
    population = ospsuite.create_population(species, n=50)
    results = model.run_simulation(population)
    
    # 4. Extract PK parameters
    pk_summary = {
        'AUC': results.calculate_auc(),
        'Cmax': results.calculate_cmax(),
        'Tmax': results.calculate_tmax(),
        'CL': results.calculate_clearance(),
        'Vdss': results.calculate_volume(),
        'F': results.calculate_bioavailability()
    }
    
    # 5. Cross-reference with ToxRefDB (safety margin)
    safety_margin = query_toxrefdb(compound_data['smiles'], pk_summary['Cmax'])
    
    return {
        'pk_profile': results.get_plasma_concentration(),
        'pk_parameters': pk_summary,
        'safety_flag': safety_margin < 10  # arbitrary threshold
    }
```
